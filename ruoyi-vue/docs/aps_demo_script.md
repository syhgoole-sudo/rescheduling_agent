# APS MVP 端到端演示脚本

## 0. 演示目标

演示“插单扰动下的动态重调度决策闭环”：从基础数据和初始调度开始，到插单影响分析、策略推荐、局部重调度、KPI 与甘特图对比、解释报告和人工确认。

## 1. 启动 Redis

操作路径：

在本机 Docker 中启动 Redis 7，端口为 `localhost:6379`。

预期现象：

Redis 容器正常运行，RESP.app 可以连接 `local-redis`。

演示讲法：

Redis 用于支撑若依系统的登录会话和缓存能力，不参与调度算法计算。

## 2. 启动 Python FastAPI 调度服务

操作路径：

```powershell
cd C:\Users\admin\Desktop\rescheduling_agent\ruoyi-vue\aps-algorithm-service
python -m uvicorn app.main:app --reload --port 8000
```

预期现象：

控制台显示 Uvicorn 启动成功，接口地址为 `http://localhost:8000`。

演示讲法：

Python 服务负责调度算法计算，包括初始调度和局部重调度。Java 后端只通过 HTTP 调用算法服务，不直接在 Java 中写排产算法。

## 3. 启动若依后端

操作路径：

```powershell
cd C:\Users\admin\Desktop\rescheduling_agent\ruoyi-vue
mvn -pl ruoyi-admin spring-boot:run
```

预期现象：

若依后端启动成功，监听默认端口。

演示讲法：

Java 后端负责业务流程编排、数据库读写、权限控制、方案版本管理和调用 Python 算法服务。

## 4. 启动若依前端

操作路径：

```powershell
cd C:\Users\admin\Desktop\rescheduling_agent\ruoyi-vue\ruoyi-ui
npm run dev
```

预期现象：

前端启动成功，浏览器访问若依登录页。

演示讲法：

前端基于若依 Vue，提供基础数据维护、调度操作、甘特图和对比分析页面。

## 5. 登录系统

操作路径：

打开前端地址，使用若依管理员账号登录。

预期现象：

进入系统首页，可以看到 APS 调度系统菜单。

演示讲法：

系统复用若依的用户、权限、菜单和日志能力，APS 是在若依框架上扩展的业务模块。

## 6. 查看基础数据

操作路径：

进入 APS 菜单，依次查看产品管理、工艺路线工序、设备组管理、设备管理、订单管理。

预期现象：

可以看到产品、设备组、设备、工艺路线和订单测试数据。

演示讲法：

调度算法依赖三类核心数据：订单需求、产品工艺路线、设备资源。MVP 中订单直接作为调度对象，不单独建立 lot 表。

## 7. 生成初始调度方案

操作路径：

进入“调度方案”页面，点击“生成初始调度”。

预期现象：

系统生成一条 `INITIAL` 方案，方案状态为当前有效状态，并保存 KPI。

演示讲法：

Java 后端查询 `NORMAL + NEW` 的普通订单，组装订单、工艺路线和设备数据，调用 Python 初始调度接口。Python 返回任务列表后，Java 保存方案主表和任务明细。

## 8. 查看初始方案任务

操作路径：

在调度方案列表中点击“查看任务”。

预期现象：

跳转到调度任务页面，并按 `planId` 过滤显示该方案下的任务明细。

演示讲法：

任务明细表保存每道工序的设备、开始时间、结束时间和状态，是甘特图和方案追溯的基础。

## 9. 查看初始甘特图

操作路径：

在调度方案页面点击“查看甘特图”。

预期现象：

进入甘特图页面，按设备维度展示任务时间条。

演示讲法：

甘特图横轴是时间，纵轴是设备，可以直观看到每台设备的负荷和任务安排。

## 10. 创建插单事件

操作路径：

进入“插单事件”页面，点击“创建并分析插单”，输入插单订单 ID。

预期现象：

系统创建插单事件，并自动完成影响范围分析。

演示讲法：

插单订单已经在订单表中维护，类型为 `INSERT`。系统会找到当前 ACTIVE 的初始方案，并以此为原方案进行分析。

## 11. 查看影响分析结果

操作路径：

在插单事件列表点击“查看分析”。

预期现象：

弹窗显示影响等级、受影响任务数、受影响订单数、受影响设备数和影响原因。

演示讲法：

系统根据插单产品工艺路线识别需要占用的设备组，再从原方案中找出插单释放时间之后、同设备组、未冻结的任务作为受影响任务。

## 12. 推荐局部重调度策略

操作路径：

点击插单事件行中的“推荐策略”。

预期现象：

弹窗显示策略类型、策略名称、冻结任务数、可调整任务数、插单任务数和推荐原因。

演示讲法：

策略推荐不是直接排产，而是根据影响等级和任务范围决定采用右移、局部重调度或插单优先的局部重调度。

## 13. 生成局部重调度方案

操作路径：

点击“生成重调度”或“生成局部重调度方案”。

预期现象：

系统生成 RESCHEDULE 新方案，并返回新方案 ID、任务数和 KPI。

演示讲法：

Java 后端把任务划分为冻结任务、可调整任务和插单任务，调用 Python 局部重调度接口。新方案保存完整任务集合，不覆盖原方案任务。

## 14. 查看 RESCHEDULE 新方案

操作路径：

回到“调度方案”页面。

预期现象：

列表中出现 `RESCHEDULE` 类型的新方案。

演示讲法：

系统通过 `source_plan_id` 记录新方案来自哪个原方案，通过 `event_id` 记录由哪个插单事件触发。

## 15. 查看 KPI 对比

操作路径：

在 RESCHEDULE 方案行点击“方案对比”。

预期现象：

弹窗显示原任务数、新任务数、变更任务数、延期订单数、延期分钟、最大延期、makespan 和插单订单完工情况。

演示讲法：

KPI 对比帮助调度员量化新方案的收益和代价，判断是否值得采用。

## 16. 查看新旧甘特图对比

操作路径：

在 RESCHEDULE 方案行点击“甘特图对比”。

预期现象：

页面展示原方案甘特图和新方案甘特图，新方案中可区分插单任务、变更任务和冻结任务。

演示讲法：

甘特图对比用于直观说明插单对原计划造成了哪些时间和设备层面的变化。

## 17. 生成解释报告

操作路径：

进入“插单事件”页面，针对已生成重调度方案的事件点击“解释报告”。

预期现象：

弹窗展示插单概述、影响范围、策略说明、重调度结果、KPI 对比、方案收益、方案代价和调度建议。

演示讲法：

解释报告当前使用规则模板生成，不接入外部大模型。它不参与排产，只把影响、策略、KPI 和方案取舍解释给调度员。

## 18. 采用或拒绝方案

操作路径：

在“调度方案”页面针对 RESCHEDULE 方案点击“采用方案”或“拒绝方案”。

预期现象：

采用后，新方案变为 `CONFIRMED`，原方案变为 `HISTORY`，插单事件变为 `CONFIRMED`。拒绝后，新方案变为 `REJECTED`，原方案恢复为 `ACTIVE`，插单事件变为 `REJECTED`。

演示讲法：

系统最终保留人工决策环节，调度算法只提供方案和依据，是否采用由调度员确认。

## 附录 A. 启动命令速查

### 启动 MySQL

本项目默认使用本机 MySQL：

- host：`localhost`
- port：`3306`
- database：`ry-vue`
- username：`root`
- password：`1234`

如果 MySQL 已安装为 Windows 服务，可在服务管理器中启动，或使用管理员 PowerShell：

```powershell
net start MySQL80
```

如果本机服务名称不是 `MySQL80`，请在 Windows 服务列表中查看实际服务名。

### 启动 Redis

推荐使用 Docker 启动 Redis 7：

```powershell
docker run -d --name local-redis -p 6379:6379 redis:7
```

如果容器已存在但未启动：

```powershell
docker start local-redis
```

### 启动 Python FastAPI 算法服务

```powershell
cd C:\Users\admin\Desktop\rescheduling_agent\ruoyi-vue\aps-algorithm-service
python -m uvicorn app.main:app --reload --port 8000
```

如遇本地依赖异常，可先执行：

## 附录 B. 核心页面检查清单

演示前建议确认以下页面均可打开：

- 产品管理：查看产品基础数据。
- 订单管理：查看 NORMAL 普通订单和 INSERT 插单订单。
- 调度方案：生成初始调度、查看 RESCHEDULE 方案、执行方案对比、采用或拒绝方案。
- 调度任务：按 `planId` 查看任务明细。
- 单方案甘特图：查看单个方案的设备维度任务条。
- 插单事件：创建插单事件、查看事件状态。
- 影响分析结果：查看影响等级、受影响任务数、受影响订单数、受影响设备数。
- 策略推荐结果：查看策略类型、冻结任务数、可调整任务数和插单任务数。
- KPI 对比：查看新旧方案延期、完工时间和任务变更指标。
- 新旧甘特图对比：查看原方案与 RESCHEDULE 方案的任务变化。
- 解释报告：查看规则模板生成的调度说明。
- 采用 / 拒绝方案：完成调度员人工决策闭环。
